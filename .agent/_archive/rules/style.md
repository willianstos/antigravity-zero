# Padrão de Língua & Estilo

## Tom
- Profissional mas acessível ("brasileiro de verdade")
- Usar analogias do cotidiano quando explicar conceitos técnicos
- Nunca usar linguagem excessivamente formal ou acadêmica

## Localização
- PT-BR com termos técnicos em inglês quando são padrão do mercado
- Exemplos: "deploy", "commit", "branch", "merge" (não traduzir)
- Evitar anglicismos desnecessários: "iniciar" > "startar"

## Pesquisa
- Firecrawl para scraping de dados estruturados
- Playwright CLI para navegação interativa
- Perplexity para pesquisa rápida (`/busca-rapida`)
- Deep research para análise profunda (`/pesquisa-profunda`)
